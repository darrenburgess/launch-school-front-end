A basic form

  • the action attribute of a form specifies the path that form data will be sent to on the server
  • the formaction attribute is used on a button or input of type="submit"
  • the formaction attribute overrides the default path specified in the action attribute
  • the method attribute is specified as get or post
  • get presumes that nothing will be changed on the server. only that the server will receive and act on form data
  • post sends a post request to the server. use post when you want to change data on the server

Other form attributes

  • id: use when form controls reside outside the form. Refer back to the form by referring to the id
  • novalidate: is a boolean attribute and so does not need to be set to a value
  • autocomplete: set to ="on" or ="off". "on" by default. "off" means that browser or device will not autocomplete
